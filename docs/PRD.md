# OKX 永续合约多周期形态筛选助手 PRD

## 项目目标

开发一个仅用于“筛选、展示、监控、提醒”的工具型系统，帮助用户从 OKX 永续合约中自动筛选符合特定均线形态的币种，并在前端统一展示结果。

当前阶段明确不做：

- 自动交易
- 自动下单
- 自动平仓止盈止损
- 用户资产接入
- 私有交易接口签名下单

## 核心规则

筛选对象：

- OKX 永续合约（`SWAP`）

筛选周期：

- `15m`
- `1H`
- `2H`
- `3H`
- `4H`

说明：

- 若 OKX 公共接口没有稳定 `3H` K 线，则使用 `1H` 本地聚合 `3H`

筛选条件：

1. 条件 A：`MA5` 与 `MA20` 收拢
2. 条件 B：`MA5` 抬头向上
3. 条件 C：价格上穿 `MA5`

默认命中逻辑：

- `A and B and C`

扩展配置：

- `A and B`
- `B and C`
- `A only`

## 展示要求

- 单页面应用
- 所有结果统一在一张表中展示
- 表格支持筛选、排序、分页、查看图表
- 图表显示最近 80 根 K 线，并叠加 `MA5 / MA20`
- 图表优先使用已完成 K 线做信号判断，未完成 K 线只作展示

## 监控与提醒

- 支持手动刷新
- 支持默认每 15 分钟自动扫描
- 仅对新信号提醒
- 同一 `instId + bar + last_closed_ts` 不重复提醒
- 支持微信机器人 / 钉钉机器人 / 通用 Webhook

## 当前实现说明

当前仓库仍采用：

- 前端：`React + TypeScript + Vite`
- 后端：`Node.js + Express + TypeScript`

这和最初建议的 `FastAPI + Python` 不完全一致，但当前版本优先保证了本地可运行性、行情拉取、统一表格展示和提醒链路。后续如果需要，我可以继续把后端迁移成 `FastAPI` 版本。
